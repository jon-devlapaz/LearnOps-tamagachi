# Theta State (canonical evidence posture)

## Summary

`docs/theta/state.md` (last updated 2026-04-11) is the canonical evidence posture for product-science claims. Sets the binding posture: Socratink is a retrieval-centered learning loop, not a general AI tutor or sleep-consolidation system. AI is a support layer for ingestion, routing, feedback, and friction reduction; AI is not itself evidence that learning occurred.

**Verified product loop** documented from a Sherlock code audit of `app.js`, `graph-view.js`, `store.js`, `main.py`, `ai_service.py`. Concept creation stores `graphData`; nodes begin mostly locked; core thesis is always first available; cold attempt is open/exploratory/unscored; non-attempt → scaffold; substantive cold attempt → `primed` + study unlock; spacing + interleaving gate re-drill; non-solid → `drilled` + gap metadata + backoff; solid → `solidified` + reward. `actualized`/hibernation is legacy/future-facing, not live.

**Core Claim Ratings** by confidence:
- High: passive-review illusion-of-competence; Generation Before Recognition as hard product constraint; cold attempts defensible when unscored + corrective study; spaced retrieval as `solidified` basis.
- Medium-high: immediate specific feedback after generation; targeted study compares attempt vs mechanism; traversal after `primed` for spacing/interleaving; non-solid → repair + delay (not retry loops).
- Medium: minimal scaffolding after non-attempt; graph supports orientation/relational encoding; sleep supports memory consolidation.
- Low / product hypotheses: 5-min re-drill delay optimal; 3-probe cap optimal; reward animation improves retention; AI scoring safe across phrasing/dialect/accessibility; `actualized` as verified state; graph topology alone proves understanding.

**Phase grounding** table maps each code phase (1-12) to its neuroscience mechanism, confidence, and product constraint.

**Recent decisions (2026-04-11)**: upgraded Generation Before Recognition + cold-attempt + spaced retrieval to high confidence per code audit + research; treat traversal after `primed` as defensible when framed as interleaving (not mastery); downgrade `actualized`/hibernation from core product claim to future hypothesis; keep AI-evaluation claims low confidence until validated.

Open scientific questions: AI-scoring trust across language/dialect/accessibility; exact feedback shapes that repair without lowering bar; spacing schedule for MVP vs long-term; reward patterns supporting persistence without false signals; whether graph topology improves transfer beyond navigation; what live retention measure justifies `actualized`.
